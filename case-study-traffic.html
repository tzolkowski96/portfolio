Case Study

Traffic Accident Analytics Engine

Analyzing 7.7 million US traffic accident records for public safety insights

Role End-to-end development
Stack Python, Pandas, Scikit-learn, Folium
Timeline Personal project

The Problem

Traffic accidents kill tens of thousands of Americans every year. The data exists—7.7 million records across 49 states from 2016 to 2023—but it's too massive and messy for most tools to handle. The question: can we find patterns that actually help predict and prevent accidents?

The Approach

I built a pipeline to process the entire dataset without choking. Memory optimization knocked usage down by 3.25%—small percentage, but when you're working with millions of rows, that's the difference between running and crashing.

The analysis had three layers:

01

Statistical Validation

Chi-square tests for categorical relationships. ANOVA for comparing means across groups. No hand-waving about whether patterns are real—statistical proof or it didn't happen.

02

Machine Learning

Random Forest classifier for severity prediction. Hit 78% accuracy predicting whether an accident would be severe. Feature importance revealed weather and road conditions matter more than time of day.

03

Geospatial Analysis

Interactive mapping with Folium to visualize hotspots. Some clusters were obvious (major intersections). Others weren't—rural stretches with unexpectedly high severity rates.

Key Findings

The data told a few stories that weren't obvious:

Weather conditions had a nonlinear relationship with severity. Light rain increased accidents but not severity. Heavy rain reduced total accidents (people stayed home) but the ones that happened were worse.

Weekend nights weren't the deadliest time—Tuesday and Wednesday rush hours were. More cars, more fatigue, more severity.

Rural roads in certain states had severity rates 40% higher than urban areas with similar traffic volumes. Infrastructure gaps, longer emergency response times.

Visual Analysis

Accident Severity Distribution

Severity 2 (moderate) accounts for the majority of incidents. Severity 4 (severe) is rare but has outsized impact on traffic flow.

Temporal Patterns

Tuesday-Wednesday rush hours showed higher severity rates than weekend nights, contrary to expectations.

Weather Impact on Accidents

Chi-square test (χ² = 96,229.39, p < 0.001) confirmed significant relationship between weather and accident severity.

Technical Decisions

Some choices that made the project work:

Chunked processing instead of loading everything into memory. Read 100k rows, process, aggregate, repeat. Let me run on a normal laptop instead of needing cloud compute.

Feature engineering created 15 derived variables from the raw data. Time-since-sunrise, days-since-last-precipitation, road-segment-historical-rate. These engineered features outperformed raw inputs in the model.

Validation strategy used spatial holdout—trained on some states, tested on others. Prevented data leakage from geographic clustering and proved the model generalizes.

Results

7.7M Records processed
78% Severity prediction accuracy
49 States covered
-3.25% Memory optimization

What I'd Do Differently

If I rebuilt this today, I'd add real-time data ingestion. The dataset was static—a snapshot. A production system would pull live accident reports and update predictions continuously.

I'd also build a proper API layer instead of just Jupyter notebooks. Make the predictions accessible to anyone who wants to use them.
